<p><em>Buy &amp; sell or auction &amp; bid (unique) collectibles on the blockchain</em></p>
<p>Non fungible tokens (NFTs) represent ownership
over digital or physical assets. Examples:</p>
<ul>
<li>Physical property -- houses, unique artwork</li>
<li>Virtual collectibles -- unique pictures of kittens, collectible cards</li>
<li>"Negative value" assets -- loans, burdens and other responsibilities</li>
</ul>
<p>In general, all houses are distinct and no two kitties are alike. Non fungible tokens (NFTs) are
<em>distinguishable</em> and must track the ownership of each one separately.</p>

<p>Every non fungible token (NFT) is identified by a unique <code>uint256</code> ID inside the contract e.g.</p>
<div class="highlight highlight-source-solidity"><pre><span class="pl-c">/// @dev An array containing the Kitty struct for all Kitties in existence. The ID</span>
<span class="pl-c">///  of each cat is actually an index into this array. Note that ID 0 is a negacat,</span>
<span class="pl-c">///  the unKitty, the mythical beast that is the parent of all gen0 cats. A bizarre</span>
<span class="pl-c">///  creature that is both matron and sire... to itself! Has an invalid genetic code.</span>
<span class="pl-c">///  In other words, cat ID 0 is invalid... ;-)</span>

Kitty[] kitties;</pre></div>
<p>The number SHALL NOT change for the life of the contract.
The pair (contract address, <code>uint256</code> tokenId) will then be a globally unique and fully-qualified identifier
for a specific asset on the ethereum blockchain.</p>
<p>Note: While some contracts may find it convenient to start with ID 0 and simply add one for each new token, callers SHALL NOT assume that ID numbers have any specific pattern,  and MUST treat the ID as a "black box". The choice of uint256 allows a wide variety of applications because UUIDs and SHA3 hashes are directly convertible to <code>uint256</code>.</p>

<p><strong>Game concept</strong></p>
<p>Player owns fishes (those are in fact ERC-721 crypto tokens) each with its unique name, image and
characteristics defined by type. Core fish parameter is weight that defines its size and influence. Main
target is to grow this parameter to become the biggest fish in the ocean.
Fish is able to gain weight only by attacking other fishes, biting off them in case of win.</p>
<p>The more powerful characteristics and the more weight the fish has the
more chance it to win a fight with another fish it represents.</p>
<p>These fishes can be traded or exchanged between players on central fish market.
Every operation in the game is launched and controlled by a contract script.</p>
<p><strong>The Fish</strong></p>
<p>Every crypto fish token has its own set of characteristics:</p>
<ul>
<li>Weight - Describes the size of the fish</li>
<li>Power - Damage on attack</li>
<li>Agility - Defense on attack</li>
<li>Speed - Cooldown time before attacks</li>
</ul>
<p>Weight is dynamic characteristic and can be changed within time while Power, Agility and
Speed are called initial parameters those are given to fish from the very moment of creation
and can't be changed in time.</p>
<p><strong>Rarity</strong></p>
<p>There are four kinds of rarity assessment  for fishes determined by sum of all
initial fish characteristics:</p>
<ul>
<li>Common</li>
<li>Rare (10+)</li>
<li>Legendary (15+)</li>
<li>Epic (25+)</li>
</ul>
